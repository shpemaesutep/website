1. The Foundation (React Native + Expo)

This app is built with React Native and Expo. Expo provides the tooling for fast iteration (Expo Go), native builds (EAS), and libraries for device features.

✅ What this gives us
  • One codebase for iOS + Android
  • Cloud builds with EAS
  • Simple push/local notification APIs
⚠️ What to be careful with
  • Managing secrets (API keys)
  • Config drift between local dev and builds
  • Platform-specific behavior (iOS vs Android notifications)

2. File Structure (Expo Router)

The app uses Expo Router. Files inside the app/ directory map directly to screens/routes.

What is a Layout (_layout.tsx)?

Layout files wrap child routes. The tabs layout defines the bottom navigation, icons, and which screens appear as tabs.

3. How the App Works (Runtime Flow)

Startup

  • App loads root layout (app/_layout.tsx).
  • Tabs mount, then Home screen fetches events.

Events (Read Path)

  • Home screen requests events from the Google Calendar API.
  • Events render as cards/list items; each includes actions like save/remind.

Saved Events & Reminders (Write Path)

  • Saving persists event IDs in AsyncStorage.
  • A local notification is scheduled (device-level reminder).

Game & Leaderboard

  • Game loop runs at ~60fps with a custom engine.
  • When the run ends, score is validated and potentially written to Firestore.
Developer note

Keep read paths fast and resilient: cache where possible, show loading/empty states, and handle API failures without crashing.

4. Local Setup (Developer Onboarding)

Prereqs

  • Node.js LTS + a package manager (npm/pnpm/yarn)
  • Expo CLI (via npx is fine)
  • Xcode (iOS simulator) and/or Android Studio (Android emulator)

Install & Run

# install dependencies npm install # start Metro npx expo start # run on iOS sim (macOS) i # run on Android emulator a
Common pitfall

If Metro starts but the app fails to load, clear caches: Ctrl + C then run npx expo start -c.

5. Configuration & Secrets

This project uses external services (Google Calendar API, Firebase). Avoid hardcoding secrets in the repo.

Do not
  • Commit real API keys to GitHub
  • Paste secrets into documentation screenshots

Tip: Using EXPO_PUBLIC_* exposes values to the client bundle. Treat them as “public-ish.” Use Firestore Rules to protect writes.

6. Google Calendar API (Events Source)

Events are fetched from a SHPE/MAES Google Calendar using the Calendar API.

Where it happens

  • Fetch runs when Home mounts (typically in app/(tabs)/index.tsx).
  • Normalize events into a UI-friendly shape (title, start time, location, id).
Rate limits & caching

If members open the app a lot, consider caching the latest successful event response in AsyncStorage and using it as a fallback when the API fails.

7. Game & Firebase Leaderboard

Leaderboard contract

  • Only store the top 5 scores globally.
  • Ask for initials only if the score qualifies for top 5.
  • Sanitize initials (length limit, alphanumeric, uppercase).

Firestore expectations

  • Reads: fetch current top 5 ordered by score desc.

9. Testing & QA

Manual QA checklist (before release)

  • Events load (cold start + after refresh)
  • Saved events persist after app restart
  • Notification permission prompt behaves correctly
  • Notification fires at expected time (set an event 2–3 minutes out)
  • Game runs smoothly and doesn’t leak memory
  • Leaderboard reads/writes work and only keeps top 5

Debugging tips

  • Test on at least one real iOS device + one real Android device before production.

10. Build & Release (EAS)

Build commands

# iOS build eas build -p ios # Android build eas build -p android

Release flow (high level)

  • Increment versions (app.json + platform version codes as needed)
  • Run QA checklist
  • Build via EAS
  • Upload to TestFlight / Play Console internal track
  • Promote to production after validation
Versioning rule

Every store submission must have a unique build number/versionCode. If Apple/Google rejects a build, you’ll still need to bump it for the next attempt.

11. Troubleshooting (Common Issues)

Metro bundler issues

  • Run npx expo start -c to clear cache.
  • Delete node_modules and reinstall if dependency tree is broken.

Events not loading

  • Verify Calendar API key is valid and enabled in Google Cloud.
  • Confirm calendar ID is correct (public calendar).
  • Check API quota / rate limits.

Notifications not firing

  • Ensure permissions granted in device settings.
  • On iOS, verify Focus/Do Not Disturb isn’t suppressing alerts.
  • Make sure scheduled time is in the future and in correct timezone.

Leaderboard issues

  • Check Firestore rules allow required read/write actions.
  • Verify the client is using the correct Firebase project config.
  • Confirm ordering/limit queries use server-side ordering.
If you see “Invalid API key” or permission errors

Do not “fix” this by hardcoding keys into the repo. Fix config injection and rules properly.
